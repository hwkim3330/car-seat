Section 01문서 개요

1.1 목적

본 문서는 차량 내 스마트 시트 모니터링(생체/자세/점유) 기능과 모바일 앱 기반 제어(원격/프리셋/프로필)를 실제 구현하기 위한 요구사항(기능/성능/인터페이스/보안/시험)을 정의한다.

문서 활용 목적

이 문서에 정의된 요구사항은 하드웨어(센서/ECU/액추에이터), 임베디드 소프트웨어, 클라우드 플랫폼, 모바일 앱 개발팀이 바로 개발 착수 가능한 수준의 기술 규격으로 활용된다.

1.2 범위

대상 기능

In-scope

  • 모니터링: 점유/승객 분류(OCS), 체압/자세, 생체(심박/호흡/HRV 등) — 가능 범위에서 단계적 적용
  • 제어: 열선/통풍, 마사지/공압(에어포켓), 볼스터, 포지션 메모리(가능 시)
  • 앱: 원격 프리셋(공조+시트), 상태 조회(정책 허용 범위), 알림, 사용자 프로필/로밍

Out-of-scope (초기 버전)

  • 의료 진단 목적 (의료기기 등급 요구되는 기능)
  • 주행 중 앱을 통한 실시간 포지션 변경 (안전 정책상 제한)

Section 02상위 사용자 시나리오 (Use Cases)

UC-01
탑승 전 원격 프리컨디셔닝

앱에서 "겨울 프리셋(22°C + 열선 2단 + 핸들열선)" 선택 → 차량 원격 시동/공조/시트 설정 자동 적용. 탑승 시 이미 최적 온도 달성.

UC-02
주행 중 운전자 피로/자세 모니터링

자세 비대칭/피로 징후 탐지 → IVI에 알림 표시 + (정책 허용 시) 미세 공압 스트레칭 동작 자동 실행으로 피로 완화.

UC-03
정차/충전 중 릴렉션/마사지

정차 조건 만족 시 앱 또는 IVI에서 릴렉션 포즈/마사지 프로그램 실행. 충전 중 최적 휴식 환경 제공.

UC-04
뒷좌석 탑승자 방치/안전 알림

점유/분류 결과 기반, 하차 후 뒷좌석 점유 탐지 시 앱 푸시 알림. 어린이/반려동물 방치 사고 예방.

Section 03시스템 아키텍처 요구사항

3.1 논리 구성요소

  • Seat Sensor Layer: Pressure map, capacitive electrode(옵션), piezo film/loadcell(옵션), IMU(옵션)
  • Seat ECU: 전처리/특징추출/추론(옵션)/액추에이터 제어
  • Vehicle Integration: Gateway/BCM/IVI/TCU, CAN/LIN/Ethernet 네트워크
  • Cloud Platform: 계정/프로필/로그/리포트/OTA(옵션)
  • Mobile App: 인증/프리셋/원격제어/알림/리포트

3.2 데이터/제어 흐름 요구사항

데이터 흐름 원칙

모니터링 데이터는 로컬(Seat ECU 또는 IVI)에서 1차 처리 후, 정책에 따라 요약값만 외부 전송한다.

원격 제어 명령은 Cloud 인증/권한검증 → TCU → 차량 네트워크 → Seat ECU 경로로만 수행된다.

3.2.1 모니터링 데이터 흐름

  1. 센서 레이어 → Seat ECU: Raw 데이터 (10~100Hz 샘플링)
  2. Seat ECU 로컬 처리: 특징 추출, 점유/자세/상태 추론
  3. Seat ECU → IVI/Gateway: 요약 이벤트 (CAN/LIN 신호)
  4. TCU → Cloud: 정책 허용 시 요약값 업링크 (최소 필드만)
  5. Cloud → App: 이벤트 알림, 상태 조회 응답

3.2.2 제어 명령 흐름

  1. 사용자 → App: 프리셋 선택 또는 개별 제어 요청
  2. App → Cloud: 인증된 명령 (서명 + TTL + nonce)
  3. Cloud → TCU: 권한 검증 후 명령 전달
  4. TCU → Seat ECU: 차량 네트워크를 통한 제어 신호
  5. Seat ECU → 액추에이터: 물리 동작 실행
  6. 응답 역순 전파: 실행 결과 → Cloud → App

Section 04기능 요구사항 (Functional Requirements)

4.1 모니터링 (Seat Monitoring)

ID항목요구사항 (Shall)검증
MON-01 점유 감지 시스템은 좌석 점유 여부를 1초 이내 업데이트 주기로 산출해야 한다. 운전석 및 조수석을 기본으로 하며, 뒷좌석은 옵션으로 지원한다. 실차 착좌/이탈 반복시험
MON-02 승객 분류 시스템은 성인/어린이/CRS(카시트)/물체(가방 등) 구분을 지원해야 한다. 초기 버전은 성인/비성인 2단계 허용. 라벨 데이터셋 정확도 ≥90%
MON-03 체압/자세 점수 시스템은 16×16 이상 체압 분포 기반으로 자세 점수(posture_score, 0-100)를 실시간 산출해야 한다. 기준자세 대비 오차 ±10점
MON-04 비대칭 탐지 시스템은 좌우 체압 비대칭 임계치(기본 20%) 초과 30초 지속 시 이벤트를 생성해야 한다. 시나리오 테스트
MON-05 심박 옵션 시스템은 무구속 방식(capacitive/BCG)으로 심박(HR) 추정치를 BPM 단위로 제공해야 한다. 정확도: 기준 ECG 대비 ±5 BPM (안정 시). 기준 ECG 비교시험
MON-06 호흡 옵션 시스템은 호흡수(RR, 분당 호흡수)를 추정해야 한다. 정확도: ±2 RPM (안정 시). 기준 센서 비교시험
MON-07 상태 추론 시스템은 졸음/피로/스트레스 상태를 신뢰도(confidence, 0-1)와 함께 산출해야 한다. 최소 3개 상태 구분. 모델 F1-Score ≥0.75

4.2 제어 (Seat Actuation)

ID항목요구사항 (Shall)검증
ACT-01 열선 제어 시스템은 열선 0~3단(OFF/Low/Medium/High)을 제어할 수 있어야 한다. 각 단계의 목표 온도: Low 30°C, Med 35°C, High 40°C (±2°C). 기능시험 + 온도추종시험
ACT-02 통풍 제어 시스템은 통풍 0~3단(OFF/Low/Medium/High)을 제어할 수 있어야 한다. 풍량 단계별 차이 체감 가능. 기능시험 + 풍량측정
ACT-03 공압 채널 시스템은 공압(에어포켓) 7채널(lumbar upper/lower/left/right, side left/right, cushion)을 0-100% 독립 제어해야 한다. 압력추종시험 (목표 대비 ±5%)
ACT-04 마사지 프로그램 시스템은 최소 4개 마사지 패턴(Wave/Pulse/Stretch/Knead)을 제공하며, 각 패턴은 3단계 강도 조절을 지원해야 한다. 시나리오시험 (패턴/강도별)
ACT-05 주행모드 연동 Comfort/Sport/Eco/Custom 주행모드 전환 시, 볼스터 지지·포지션·공압이 모드별 프리셋으로 자동 전환되어야 한다. CAN 입력 연동시험
ACT-06 안전 제약 주행 중(차속 > 5km/h) 포지션(전후/높이/틸트) 변경은 차단, 리클라인은 25° 제한. 주행 중 stretch 강도 3은 금지. 정책시험 (차속별 동작확인)

4.3 앱/커넥티드 (Mobile App)

ID항목요구사항 (Shall)검증
APP-01 계정/차량 페어링 앱은 사용자 계정과 차량을 소유권 기반(VIN 검증)으로 페어링해야 한다. 1계정 다차량 지원. 계정/권한시험
APP-02 원격 프리셋 앱은 공조+시트 설정을 프리셋으로 저장/호출해야 한다. 프리셋 생성은 3단계 이하(선택→강도→저장). E2E시험
APP-03 원격 시동 연동 원격 프리셋은 원격 시동/공조 실행과 결합될 수 있어야 한다. 프리컨디셔닝 목표 온도 도달 예상시간(ETA) 표시. E2E시험
APP-04 상태 조회 앱은 좌석 점유/시트 모드/열선·통풍 상태를 조회할 수 있어야 한다. 조회 결과는 10초 이내 갱신. API시험
APP-05 알림 시스템은 이벤트(뒷좌석 점유, 자세 경고, 온도 이상 등)를 앱 푸시 알림으로 통지해야 한다. 알림에는 원인/추천조치/차량상태링크 포함. 이벤트 주입시험
APP-06 프로필 로밍 사용자 프로필(시트 포지션/공압/마사지/기후 설정)은 서버에 저장되어 재로그인 시 복원되어야 한다. 다른 차량에서도 적용 가능. 서버 연동시험
요구사항 집계

모니터링 7개 (MON-01~07, 옵션 2개 포함) + 제어 6개 (ACT-01~06) + 앱 6개 (APP-01~06) = 총 19개 기능 요구사항

Section 05비기능 요구사항 (Non-Functional)

5.1 성능 (Performance KPI)

NFR-PERF-01
원격 명령 응답

원격 명령 요청 → 적용 결과 회신까지 p95 ≤ 5초 (네트워크 상황에 따라 조정 가능)

NFR-PERF-02
이벤트 탐지 지연

점유/자세 이벤트 탐지 지연 ≤ 1초 (로컬 Seat ECU 기준)

NFR-PERF-03
자동 복구

센서/네트워크 일시 오류 시 자동 재시도/페일세이프 복구 (30초 이내)

5.2 신뢰성 / 내구성

NFR-REL-01
캘리브레이션

차량 진동/온도/습도 환경에서 센서 드리프트를 고려한 자동 캘리브레이션 절차 제공

NFR-REL-02
페일세이프

ECU 결함 시 시트를 안전 상태로 복귀 (열선 차단, 펌프 정지, 포지션 동결)

NFR-REL-03
내구 수명

센서 매트릭스 10만 회 이상 착좌/이탈 사이클에서 특성 유지

NFR-UX-01
프리셋 생성

프리셋 생성은 3단계 이하 (선택 → 강도 → 저장)

NFR-UX-02
안전 UI

주행 중 위험 동작은 UI에서 비활성 표시 + 경고 메시지 처리

NFR-UX-03
반응 시간

앱 내 UI 조작(탭/전환) 반응시간 ≤ 200ms

Section 06인터페이스 요구사항 (Interface)

6.1 차량 네트워크

ID요구사항
IF-VNET-01Seat ECU ↔ Gateway/BCM 간 제어 메시지는 CAN/LIN 신호로 정의되어야 한다 (신호명/주기/스케일/오프셋 포함).
IF-VNET-02주행모드/차속/기어/도어 상태 등 안전 정책 판단 신호를 수신해야 한다.
IF-VNET-03시트 상태(열선/통풍/포지션)는 100ms 주기 이하로 차량 네트워크에 publish되어야 한다.

6.2 클라우드 API

ID요구사항
IF-API-01원격 제어 API는 명령 ID, 요청시간, 유효기간(TTL), 사용자/차량 식별자, 디지털 서명을 포함해야 한다.
IF-API-02이벤트 업링크는 최소 필드(시간, 차량ID, 좌석, 이벤트타입, 심각도, 요약값)만 전송해야 한다.
IF-API-03API 인증은 JWT(Access + Refresh Token) 방식을 사용하며, Access Token TTL은 60분 이하.

6.3 모바일 앱

ID요구사항
IF-APP-01푸시 알림에는 "원인 / 추천 조치 / 차량 상태 링크"가 포함되어야 한다.
IF-APP-02앱은 iOS 15+ / Android 12+를 지원해야 하며, 태블릿 레이아웃도 지원해야 한다.
IF-APP-03앱은 오프라인 모드에서 캐시된 프리셋/프로필 목록을 표시할 수 있어야 한다 (PWA 지원).

Section 07보안 / 개인정보 요구사항

Security by Design

원격 제어 기능은 차량 안전과 직결되므로, 설계 단계부터 보안을 내재화해야 한다.

ID항목요구사항 (Shall)
SEC-01 강한 인증 원격 제어는 강한 사용자 인증(MFA 또는 디바이스 바인딩 등 OEM 정책) 후 허용되어야 한다.
SEC-02 명령 무결성 명령 메시지는 서명/무결성 검증을 거쳐야 한다. 리플레이 방지: nonce + TTL(최대 5분) 적용.
SEC-03 생체 데이터 보호 생체 관련 데이터는 기본적으로 로컬 처리 우선. 서버 전송 시 명시적 동의 + 최소수집 원칙 적용.
SEC-04 동의 철회 사용자 동의 철회 시 데이터 삭제/수집 중지 정책을 제공해야 한다 (GDPR/PIPA 준수).
SEC-05 전송 암호화 Cloud-App 간 통신은 TLS 1.2 이상을 사용해야 한다.
SEC-06 접근 제어 관리자/운전자/동승자 역할 기반 접근 제어(RBAC)를 적용해야 한다.

Section 09버전 / 단계 로드맵

MVP (v1)
기본 점유/제어 + 원격 프리셋
점유/기본 자세(MON-01~04) + 열선/통풍(ACT-01~02) + 원격 프리셋/알림(APP-01~05) + 기본 보안(SEC-01~02). 센서: 16×16 압력 매트릭스. 대상: 1열(운전석+조수석).
V2
공압/마사지 + 프로필 로밍
체압 고도화 + 공압 7채널 마사지/스트레칭(ACT-03~04) + 주행모드 연동(ACT-05) + 프로필 로밍(APP-06) + 안전 정책 고도화(ACT-06). 대상: 전좌석.
V3
생체 모니터링 + AI 추론
생체(ECG/BCG/HRV)(MON-05~06) + 졸음/스트레스 AI 추론(MON-07) + 리포트/데이터 서비스 + OTA 업데이트. 의복/체형/노면 보정 자동화.

D. 설계 가정 (Assumptions)

  • 대상 좌석: 1열(운전석+조수석) 기본, 2열(뒷좌석) 옵션. MVP는 운전석 단독 시작.
  • 액추에이터 범위: 열선/통풍(MVP) + 공압 7채널(V2부터). 포지션 메모리는 차량 사양에 따라 결정.
  • 원격 제어 허용 범위: 탑승 전 프리컨디셔닝 + 정차/충전 시만 허용. 주행 중 원격 포지션 변경 불가.
  • 생체 센서: V3 단계에서 capacitive electrode 방식 적용. MVP/V2에서는 체압 기반 간접 추정.
  • 통신 환경: LTE/5G 기반 TCU 통신. 터널/지하주차장 등 음영 지역에서는 명령 큐잉 후 재전송.
